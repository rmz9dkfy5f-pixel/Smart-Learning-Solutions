Homepage wireframe
Here is the revised homepage wireframe in low-fidelity form.
1) Header
Left: logo Center/right: Home, Workshops, Programs, Resources, About, Contact Persistent button: Book a Workshop
Behavior:
* transparent or light header at top
* solid sticky header on scroll
* booking CTA always visible
2) Hero
Large premium headline: Hands-On STEM Workshops for Curious New Learners
Supporting copy: A short, polished statement that frames Smart Learning Solutions as a provider of high-quality STEM workshop experiences built around robotics, coding, physical science, and engineering exploration.
Primary CTA: Book a Workshop Secondary CTA: Explore Programs
Visual:
* one strong hero image or montage
* premium photography style, not clip-art or crowded collage
* subtle animated accent shapes inspired by robotics / engineering diagrams
Why this matters: the current homepage names the two program pillars, but it does not strongly position the site around a workshop action or premium experience. 
3) Trust strip
A slim band immediately below hero:
* Edison / robotics program reference
* Whybricks / PSTEM reference
* K–12 / age-range indicator
* possibly “engineers, coders, and educators” if validated with owner
This should be visually restrained and premium. The current About page already gives you the basis for this credibility language. 
4) “Choose Your Workshop Experience”
Two large editorial cards:
Coding with Robots Short description focused on problem-solving, logic, creativity, and interactive robotics.
PSTEM Short description focused on physical science, engineering thinking, and hands-on experimentation.
Each card includes:
* age band
* format tags
* one-line outcome
* CTA: View Program
This section preserves the strongest structure already present on the current homepage, but upgrades it into a clearer decision point. 
5) “Who It’s For”
Three-column audience selector:
* Students
* Schools / Educators
* Community Programs / Groups
Each option leads to the same workshop funnel but reframes the value proposition. This is important because the current site speaks broadly to educators and parents; the redesign should keep that flexibility while making “new students” feel central. 
6) “What Happens in a Workshop”
A clean 3-step process:
1. Choose a workshop path
2. Customize for your group
3. Book and prepare
Each step gets a short sentence. This replaces the vague current booking pattern with a more premium service model.
7) Featured workshop outcomes
A section with 4 polished benefit tiles:
* Build and test ideas
* Learn through guided exploration
* Develop coding and problem-solving skills
* Experience real STEM thinking
This is where the site starts selling the experience, not just listing products.
8) Sample activities / previews
A curated preview section pulled from your strongest existing resource themes.
Examples based on the current Downloads page:
* gears and motion
* forces and ramps
* friction
* gravity
* bridges / structures
Each preview should be short and visual, with one CTA: View Sample Activities. The current Downloads page already contains named investigation-style content that can be repackaged here far more elegantly. 
9) Premium differentiator section
A full-width section answering: Why choose Smart Learning Solutions?
Use 3–4 pillars:
* expert-led STEM experiences
* hands-on learning
* workshop formats tailored to learners
* trusted program foundations
This section should feel elevated and brand-led, with restrained typography and generous spacing.
10) Booking CTA block
A dedicated conversion section before footer: Ready to bring a STEM workshop to your students?
Include:
* primary CTA: Book a Workshop
* secondary CTA: Contact Us
* microcopy on response expectations and customization
11) Footer
Simplified footer:
* nav links
* contact details
* booking CTA
* legal / copyright
* optional social links later
The current footer repeats navigation and still shows a 2021 copyright, which undercuts trust; the redesign should clean that up completely.
