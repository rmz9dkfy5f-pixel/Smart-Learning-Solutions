Recommended minimal docs and where to place them:
* README.md — root: high-level project overview, stack, run/build instructions.
* ARCHITECTURE.md — root or /docs: information architecture, component architecture, data models, routing, state management.
* DESIGN.md or /design/README.md — /design: layout/grid, design system, component inventory, interaction patterns, accessibility guidelines.
* API.md or /docs/api.md — /docs or /api: API contracts, endpoints, request/response schemas, auth.
* CONTENT.md — /content or /docs: content strategy, CMS structure, metadata, i18n plan.
* ACCESSIBILITY.md — /docs or /checklists: semantic HTML rules, ARIA usage, keyboard & focus expectations, testing guidance.
* PERFORMANCE.md — /docs: performance budget, critical rendering path, lazy-loading rules, asset priorities.
* SECURITY.md — /docs: auth flow, input validation rules, CSP, secrets handling.
* TESTING.md — /tests or /docs: test strategy, folder conventions, CI test steps, fixtures/mocks.
* DEPLOYMENT.md — /deploy or /docs: build pipeline, CI/CD, environment vars, release process, rollback plan.
* ROADMAP.md — /docs or root: planned features, milestones, priorities.
* CHANGELOG.md — root: release notes.
* CONTRIBUTING.md — root: contribution rules, code style, branching, PR checklist.
* .github/ or .gitlab/ — CI workflows, issue/pr templates.
