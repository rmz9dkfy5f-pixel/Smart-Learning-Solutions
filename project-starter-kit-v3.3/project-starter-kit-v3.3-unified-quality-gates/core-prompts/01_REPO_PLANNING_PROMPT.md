# 01 — Repo Planning Prompt

Use this when a repo already exists or when starting a new repo.

```md
We are in Phase 1.

Inspect the repository structure.

Do not edit files.

Report:

1. current folder structure
2. existing planning/docs files
3. missing required project-control files
4. duplicate or conflicting source-of-truth files
5. likely project type
6. recommended starter files to add
7. files that should not be added yet
8. risks in the current structure
9. proposed Phase 1 documentation changes
10. approval request before any changes

Required files for this project system:

- README.md
- PROJECT_BRIEF.md
- CONTEXT.md
- STATUS.md
- PLAN.md
- PHASE_GATES.md
- BACKLOG.md
- ROADMAP.md
- DECISION_LOG.md
- SLICE_REVIEWS.md
- LESSONS_LEARNED.md
- CHANGELOG.md
- RELEASE_NOTES.md
- COMMIT_NOTES.md
- PROGRESS_NOTE.md
- PROGRESS_NOTES.md
- CLAUDE.md
- AGENTS.md

COMMIT_NOTES.md and PROGRESS_NOTE.md are GitHub-specific and must be kept unconditionally.
```
