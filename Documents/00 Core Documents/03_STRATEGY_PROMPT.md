Create `docs/STRATEGY.md` for this project.

Your job is to define the business, audience, content, and strategic truth of the project so future AI/code agents do not invent direction.

Do not write implementation code.
Do not create marketing fluff.
Do not fabricate metrics, testimonials, capabilities, pricing, or features.
Use placeholders where project-specific information is missing.
Flag missing information clearly.

# Required Output

# Strategy

## 1. Project Overview
Explain what this project is, what it does, and why it exists.

Include:
- project name
- project type
- current stage
- primary purpose
- what this project is not trying to be

## 2. Mission / Purpose
Write a concise mission statement.

Format:
> This project exists to [specific outcome] for [specific audience] by [specific method/value].

## 3. Primary Goals
List 3–5 project goals.

For each goal include:
- goal name
- desired outcome
- why it matters
- success signal

## 4. Target Audiences
Define each audience this project serves.

For each audience include:
- who they are
- what they need
- what problem they have
- what action the project should help them take
- what messaging should avoid

## 5. Core Offering / Value Proposition
Define what the project actually provides.

Include:
- primary offering
- secondary offerings
- strongest differentiator
- proof or credibility currently available
- proof or credibility still missing

## 6. Content Strategy
Define the content direction.

Include:
- tone
- voice
- reading level
- key messages
- CTA strategy
- content types
- metadata needs
- what claims should not be made

## 7. Brand / Design Direction
Define the strategic design direction.

Include:
- desired impression
- visual personality
- examples or references if available
- design qualities to avoid

## 8. Constraints
List real constraints.

Consider:
- timeline
- budget
- technical limitations
- client requirements
- brand requirements
- legal/privacy concerns
- accessibility expectations
- hosting/deployment limits

## 9. Strategic Non-Negotiables
List rules that should not be violated.

Examples:
- do not invent unsupported claims
- keep audience-specific CTAs clear
- preserve existing brand direction
- avoid unnecessary complexity
- prioritize clarity over cleverness

## 10. Open Questions
List missing decisions that block better strategy.

For each question include:
- question
- why it matters
- recommended owner/decision-maker
- whether work can continue without it

## 11. Source-of-Truth Rules
Explain how this file should be used.

Include:
- this file controls business/project direction
- content and design work should reference this file
- if strategy conflicts with implementation, flag the conflict
- do not overwrite strategic uncertainty with invented certainty

# Quality Bar

Be specific.
Be practical.
Separate facts from assumptions.
Use placeholders only when necessary.
Do not overstate what the project can do.
