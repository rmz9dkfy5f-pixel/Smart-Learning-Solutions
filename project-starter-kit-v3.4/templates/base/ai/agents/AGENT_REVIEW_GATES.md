# Agent Review Gates

## Review Gate A — Before Editing

Agent must confirm:

- Objective understood
- Relevant files inspected
- Existing work preserved
- Plan is minimal
- Validation method selected

## Review Gate B — After Editing

Agent must confirm:

- Files changed are listed
- No unrelated changes made
- Validation run or documented as unavailable
- Risks listed

## Review Gate C — Before PASS

Agent must confirm:

- Done criteria checked
- Phase gate checked
- Security/compatibility impact considered
- Next action clear

## Review Gate D — Migration Safety

For existing repos:

- Existing instructions preserved
- Existing docs preserved
- Template conflicts sent to `.v34_migration_review/`
- No forced merge without approval
