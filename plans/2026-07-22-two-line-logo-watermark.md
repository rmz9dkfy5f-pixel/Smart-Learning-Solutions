# Plan: Two-Line Logo Lockup Watermark on About Page

**Date:** 2026-07-22
**Status:** Complete

---

## Objective

Place the client's full two-line logo lockup (including the script-font "solutions" swoosh,
excluded from the single-line header/footer crop shipped in v2.26.0) as a subtle background
watermark behind the "Built on Real Expertise" mission copy on `about.html` — closing the
non-blocking open item noted in that session's `PROGRESS_NOTE.md`.

## Current State

- `about.html`'s "Built on Real Expertise" section (`.about-intro`, lines 91-161) is a two-column
  grid: left column is 3 mission paragraphs under an `<h2>`, right column is a 4-item
  `.credentials-list`. `align-items: start` means the columns are unequal height.
- No two-line lockup asset existed anywhere in the repo. The source, `pics/Logo/Logo.png.avif`
  (277×164, RGBA/alpha), is the full two-line lockup; the shipped single-line mark
  (`src/images/brand-logo-mark.png`) is a top-aligned 277×120 crop of it that explicitly excludes
  the bottom ~44px containing the swoosh.
- `about.html` carries its own page-scoped `<style>` block (head) for
  `.about-intro`/`.credentials-list`/`.credential-item`, per this repo's convention
  (`docs/workflow/claude-code-workflow.md`: no new CSS files, page-specific styles live in a page's
  own `<style>` block).
- `main.css` already has an established low-opacity decorative-background pattern
  (`.audience-photo-bg`/`.format-card-photo-bg`, opacity 0.09-0.10; `.cta-band-orb`) pairing
  `position: absolute; z-index: 0` on the decorative layer with `position: relative; z-index: 1`
  on the real content — required for correct paint order, not just style.
- ADR-017 already decided the client logo displays in native orange/teal with no CSS filter; that
  carries forward unchanged here (see ADR-018 below).
- `.about-intro p` uses `color: var(--text-muted)`, already close to the AA 4.5:1 floor
  independent of this change — meant little spare contrast margin was available, so opacity was
  deliberately set at the low end of the sitewide convention and confirmed visually rather than
  assumed.

## Assumptions

- Placement (mission-section watermark, not a CTA-band accent or a fully visible hero mark) and
  scope (repo-only, no staging deploy this pass) were both owner-confirmed before implementation.
- The two-line asset is a straight format conversion of the full, uncropped source — not a new
  crop region.

## Slice 1 — Asset export

**Goal:** Produce the two-line lockup as a web-usable asset.

**Planned edits:**
- `sips -s format png "pics/Logo/Logo.png.avif" --out "src/images/brand-logo-lockup-full.png"` —
  native 277×164 export, no crop/resize (CSS controls display size)

**Validation:**
- `sips -g` confirmed 277×164, RGBA alpha, PNG
- Visually confirmed the "solutions" swoosh is present and uncropped in the export

## Slice 2 — HTML + CSS integration

**Goal:** Place the watermark behind the mission copy without disrupting layout, legibility, or
the existing scroll-reveal animation.

**Planned edits (`about.html`):**
- Wrapped the mission column in `.about-mission` (alongside the existing `reveal-up` hook) and
  inserted an `aria-hidden` watermark `<div>` with the new image (`alt=""`, `loading="lazy"`,
  explicit `width`/`height`) as its first child
- Added CSS to the page's existing `<style>` block: `.about-mission { position: relative }`
  scopes the positioning context to the (shorter) mission column, not the full grid;
  `.about-mission-watermark` is absolutely centered, `width: 60%; max-width: 320px`,
  `pointer-events: none`; the image itself is `opacity: 0.08`; `.about-mission h2, .about-mission
  p` get `position: relative; z-index: 1` — required so the text paints above the `z-index: 0`
  watermark, not just for style consistency

**Validation:**
- Playwright check (disposable, scratchpad-only): confirmed computed styles
  (`position: absolute`, `z-index: 0`, `opacity: 0.08`, `pointer-events: none` on the watermark;
  `position: relative; z-index: 1` on `h2`/`p`), confirmed the watermark's rendered box (312×184.7
  at a 1440px viewport) stays within the mission column's bounds (`watermarkOverflowsColumn:
  false`), confirmed `aria-hidden="true"` and `alt=""`
- Visual check at 375 / 768 / 1024 / 1440px (per `docs/DESIGN.md` §9): watermark reads as
  intentionally subtle at every breakpoint, stays centered within the mission column only (no
  bleed into the `.credentials-list` cards below, including at the 768px stacking transition),
  and all three paragraphs plus the heading remain fully legible — no adjustment to opacity or
  position was needed
- Confirmed `prefers-reduced-motion: reduce` still resolves the `reveal-up` parent to
  `opacity: 1; transform: none` (the existing static fallback), unaffected by the new child

## Risks

- Opacity (0.08) and position were starting points pending a visual check, given hard-edged logo
  strokes over already-borderline `--text-muted` text — confirmed acceptable at all 4 breakpoints
  during Slice 2 validation, no further adjustment needed.
- New `.about-mission` wrapper class changes the DOM slightly; confirmed no existing selector in
  `main.css` targeted the old bare `<div>` inside `.about-intro`, so nothing else was affected.
- MINOR version bump (v2.27.0) per `docs/VERSIONING.md` §4 ("new feature or component added") —
  consistent with how the original logo placement (v2.26.0) was also MINOR, not PATCH.

## Rollback

Revert the commit; `about.html` and its `<style>` block return to the pre-change state. The new
`src/images/brand-logo-lockup-full.png` file can be deleted with no functional impact, since
nothing else references it.
